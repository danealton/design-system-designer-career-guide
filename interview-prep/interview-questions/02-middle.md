## Технические вопросы

### Продвинутый UI-дизайн

**Q: Расскажи о принципах Atomic Design. Как ты применяешь их в работе?**

**Ключевые моменты для ответа:**

- Понимание уровней: атомы, молекулы, организмы, templates, pages
- Примеры применения в реальных проектах
- Преимущества и ограничения подхода

**Q: Как ты проектируешь масштабируемую систему компонентов? Какие принципы используешь?**

**Ключевые моменты для ответа:**

- Модульность и переиспользование
- Консистентность в naming, spacing, стилях
- Примеры из опыта создания/развития систем

**Q: Расскажи о своем опыте работы с accessibility (доступностью). Какие WCAG принципы ты применял?**

**Ключевые моменты для ответа:**

- Знание WCAG 2.1 Level AA
- Конкретные примеры: контрастность, ARIA-атрибуты, клавиатурная навигация
- Кейсы из опыта

**Q: Как ты работаешь с темами (темная/светлая)? Какие подходы используешь?**

**Ключевые моменты для ответа:**

- Понимание дизайн-токенов для тем
- Подходы к проектированию (семантические токены vs прямые значения)
- Примеры реализации

### Инструменты

**Q: Расскажи о продвинутых возможностях Figma, которые ты используешь в работе с компонентами.**

**Ключевые моменты для ответа:**

- Условная логика в компонентах
- Сложные variants с множественными свойствами
- Auto-layout для адаптивности
- Плагины для автоматизации

**Q: Как ты создаешь и поддерживаешь дизайн-токены? Какой процесс используешь?**

**Ключевые моменты для ответа:**

- Структура токенов (цвета, типографика, spacing, shadows)
- Процесс создания и обновления
- Синхронизация между дизайном и кодом
- Инструменты (Style Dictionary, Figma Variables)

**Q: Как ты работаешь с разработчиками при передаче компонентов? Что важно передавать?**

**Ключевые моменты для ответа:**

- Спецификации и документация
- Понимание технических ограничений
- Процесс ревью и итераций
- Примеры успешного взаимодействия

### Комплексные компоненты

**Q: Опиши процесс создания сложного компонента (например, форма с валидацией). С чего начинаешь?**

**Ключевые моменты для ответа:**

- Анализ требований и use cases
- Разбиение на части (композиция)
- Проектирование состояний (loading, error, success)
- Edge cases и граничные ситуации

**Q: Как ты проектируешь состояния компонентов (loading, error, empty)? Приведи примеры.**

**Ключевые моменты для ответа:**

- Понимание всех необходимых состояний
- Визуальные решения для каждого состояния
- Примеры из реальных проектов
- UX-принципы для состояний

**Q: Что такое edge cases и как ты их учитываешь при проектировании компонентов?**

**Ключевые моменты для ответа:**

- Понимание граничных ситуаций
- Примеры: длинный текст, множество элементов, экстремальные размеры
- Подходы к решению

## Продуктовые вопросы

### Исследования и аналитика

**Q: Как ты анализируешь использование компонентов в продуктах? Какие метрики важны?**

**Ключевые моменты для ответа:**

- Методы анализа (опросы, аналитика, code review)
- Метрики: adoption rate (уровень внедрения), частота использования, satisfaction (удовлетворенность)
- Примеры выводов и действий на основе анализа

**Q: Как ты выявляешь пробелы в дизайн-системе? Опиши свой процесс.**

**Ключевые моменты для ответа:**

- Анализ запросов от дизайнеров/разработчиков
- Аудит существующих решений
- Приоритизация пробелов
- Примеры из опыта

**Q: Расскажи о своем опыте сбора обратной связи от пользователей дизайн-системы (дизайнеров, разработчиков).**

**Ключевые моменты для ответа:**

- Методы сбора (интервью, опросы, воркшопы)
- Как обрабатываешь обратную связь
- Примеры улучшений на основе feedback

### Проектирование систем

**Q: Как ты создаешь гайдлайны по использованию компонентов? Что в них включаешь?**

**Ключевые моменты для ответа:**

- Структура гайдлайнов (описание, примеры, do's and don'ts)
- Визуальные примеры
- Инструменты (Storybook, Zeroheight, Notion)
- Примеры из опыта

**Q: Что такое design patterns и как ты их проектируешь?**

**Ключевые моменты для ответа:**

- Понимание паттернов взаимодействия
- Примеры: формы, навигация, модальные окна
- Процесс проектирования и документирования

### Кросс-функциональная работа

**Q: Опиши свой опыт работы с разработчиками. Как вы вместе работаете над компонентами?**

**Ключевые моменты для ответа:**

- Процесс совместной работы
- Технические обсуждения (API, пропсы, события)
- Ревью кода и дизайна
- Примеры успешной коллаборации

**Q: Как ты консультируешь других дизайнеров по использованию компонентов? Приведи пример.**

**Ключевые моменты для ответа:**

- Подходы к консультированию
- Типичные вопросы и как на них отвечаешь
- Примеры ситуаций, когда помог решить проблему

**Ключевые моменты для ответа:**

- Причины рефакторинга
- Процесс: анализ, планирование, реализация
- Учет backward compatibility (обратной совместимости)
- Результаты

**Q: Как ты участвовал в планировании развития UI-kit'а? Какие улучшения предлагал?**

**Ключевые моменты для ответа:**

- Примеры предложений
- Обоснование необходимости
- Приоритизация
- Реализация и результаты
