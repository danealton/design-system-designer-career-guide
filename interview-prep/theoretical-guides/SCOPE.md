# Скоуп работ: Теоретические гайды

> Определение объема и приоритетов работ по созданию теоретических гайдов

## Критерии отбора тем

Темы отбирались на основе:

1. **Частота упоминания** в вопросах для интервью
2. **Важность для разных уровней** (Junior → Lead)
3. **Наличие в framework компетенций**
4. **Сложность темы** (нужен ли детальный разбор)

---

## Приоритет 1: Обязательные гайды

### ✅ 1. Design Tokens

**Статус:** TODO  
**Уровень:** Все уровни  
**Приоритет:** Высокий

**Почему важно:**

- Упоминается на всех уровнях (Junior → Lead)
- Базовая концепция для понимания дизайн-систем
- Связано с множеством других тем (темизация, кастомизация)

**Содержание:**

- Что такое design tokens
- Типы токенов (цвета, типографика, spacing, shadows, borders)
- Структура и организация токенов
- Семантические vs прямые токены
- Работа с токенами в Figma (Variables)
- Design tokens pipeline (от дизайна до кода)
- Инструменты (Style Dictionary, Theo, Figma Tokens)
- Best practices

**Связанные термины:** White-label, Theming (темизация), кастомизация, 8pt Grid

### ✅ 2. Accessibility (WCAG, Inclusive Design)

**Статус:** TODO  
**Уровень:** Все уровни  
**Приоритет:** Высокий

**Почему важно:**

- Критично для всех продуктов
- Требование на всех уровнях (от Level A до AAA)
- Множество вопросов в интервью

**Содержание:**

- Что такое accessibility и почему это важно
- WCAG стандарты (Level A, AA, AAA)
- Основные принципы (Perceivable, Operable, Understandable, Robust)
- ARIA-атрибуты и семантическая разметка
- Контрастность и читаемость
- Клавиатурная навигация
- Screen readers и assistive technologies
- Inclusive design vs accessibility
- Тестирование доступности
- Best practices для компонентов

**Связанные термины:** ARIA, семантика, WCAG

**Статус:** TODO  
**Уровень:** Middle+  
**Приоритет:** Средний-Высокий

**Почему важно:**
